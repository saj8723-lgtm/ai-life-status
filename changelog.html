api/og.js 新規作成 NEW

項目内容
ランタイム Vercel Edge Function + @vercel/og（ImageResponse）で 1200×630 PNG を生成
画像内容 左カラム (620px): ギルドラベル・名前・ジョブ・Lv/称号バッジ・必殺技・弱点
右カラム: レーダーチャート（6軸、絵文字+日本語ラベル付き）
決定論的生成 hash(名前+日付) + 線形合同法 PRNG sr() で、 同じ名前・同じ日は必ず同じ結果。サイトの gen() と乱数シーケンスを完全一致させLv不一致を解消。
レーダーチャート SVG polygon + line で描画。 ラベルは <text> 非対応のため HTML div を絶対配置で重ねる方式。 R=155, 6頂点すべて SVG 内に収まることを計算で確認。
フォント @vercel/og 標準CJKフォント（自動ロード）を使用。woff2非対応のため DotGothic16 の読み込みは省略。
デザイン 背景グラデーション linear-gradient(135deg,#faf5ff,#e8ddf5)、 上部レインボーバー、装飾円。サイト本体のカラーパレットと統一。

技術的つまずきと解決ログ

  • ① woff2エラー（無音の空レスポンス）
    Google Fonts から woff2 を取得 → satori が「Unsupported OpenType signature wOF2」でクラッシュ。 レスポンスは HTTP 200 だがボディが空。
    → カスタムフォント読み込みを全廃し、@vercel/og の自動CJKフォントに切り替え
  • ② ERR_REQUIRE_ESM（Node.js runtime）
    package.json が CJS / @vercel/og が ESM のため、Node.js 18/20 で import 失敗。
    → Edge runtime に戻すことで解決
  • ③ X が「Hello」画像をキャッシュ
    診断用に書いた「Hello」だけの版を X が24時間キャッシュ。
    → og:image URL に ?v=N パラメータを付けてキャッシュバスト
  • ④ LvがツイートとOGP画像でずれる
    genOG() の乱数消費回数がクライアントの gen() と不一致。
    → r() の呼び出し順をサイト本体と完全に揃える（wk・qt skip を追加）
  • ⑤ <text> ノード非対応
    satori は SVG の <text> を非サポート。レーダーチャートのラベルが全てエラーに。
    → HTML <div> を position:absolute で SVG の上に重ねる方式に変更
  • ⑥ ✦ ◆ などの記号が文字化け
    @vercel/og 標準フォントにはグリフが含まれない記号は □ になる。
    → 装飾文字を全削除し、日本語テキストのみに
  • ⑦ 弱点テキストが欄をはみ出す
    whiteSpace:nowrap + 24px フォントでは最長テキストが 600px 超に。
    → カラム幅を 620px に拡張、フォントを 20px に調整（計算で全文字列1行収まりを確認）
